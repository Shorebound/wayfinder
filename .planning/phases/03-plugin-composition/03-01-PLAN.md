**1. Create `engine/wayfinder/src/core/TopologicalSort.h`:**

A standalone, header-only Kahn's algorithm in `namespace Wayfinder`:

```cpp
struct TopologicalSortResult
{
    std::vector<size_t> Order;      // indices in topological order
    bool HasCycle = false;
    std::string CyclePath;          // human-readable cycle description (empty if no cycle)
};

[[nodiscard]] inline auto TopologicalSort(
    size_t nodeCount,
    std::span<const std::vector<size_t>> adjacency,
    std::function<std::string(size_t)> nameOf = {}
) -> TopologicalSortResult;
```

- Takes node count, adjacency list (adjacency[i] = list of nodes that depend on i, i.e. edges from i -> dependents), and an optional name function for cycle path reporting.
- Computes in-degree from the adjacency list, runs Kahn's algorithm.
- If `Order.size() != nodeCount`, sets `HasCycle = true` and builds `CyclePath` from residual nodes (reuse the cycle-extraction logic currently in SubsystemRegistry::ExtractCyclePath).
- Includes: `<vector>`, `<span>`, `<string>`, `<functional>`, `<queue>`, `<format>`.
- Namespace: `Wayfinder`.

**2. Create `engine/wayfinder/src/app/SubsystemManifest.h`:**

Template class `SubsystemManifest<TBase>` in `namespace Wayfinder` that owns the runtime subsystem instances. It receives the entry data from SubsystemRegistry::Finalise().

Public API:
- `[[nodiscard]] auto Initialise(EngineContext& context, const CapabilitySet& effectiveCaps) -> Result<void>` - creates instances in topological order, capability-gated.
- `void Shutdown()` - destroys in reverse topological order.
- `template<typename T> auto Get() -> T&` - asserts if not found/active.
- `template<typename T> auto Get() const -> const T&` - const overload.
- `template<typename T> auto TryGet() -> T*` - returns nullptr if not found/active.
- `template<typename T> auto TryGet() const -> const T*` - const overload.
- `template<typename T> auto IsRegistered() const -> bool` - check if type is registered.

Internal representation - same data as SubsystemRegistry::Entry but owned by the manifest:
- `std::vector<Entry> m_entries` (ConcreteType, AbstractType, Descriptor, Factory, Instance, DebugName, Active)
- `std::unordered_map<std::type_index, size_t> m_typeToIndex`
- `std::unordered_map<std::type_index, std::type_index> m_abstractRedirect`
- `std::vector<size_t> m_initOrder`

The manifest is move-only (deleted copy, default move). Constructed by SubsystemRegistry::Finalise() via a private constructor + friend.

The Initialise/Shutdown/Get/TryGet/IsRegistered logic is moved verbatim from SubsystemRegistry's current implementation.
