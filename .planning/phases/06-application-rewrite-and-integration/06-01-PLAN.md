  <action>
    Create SDLWindowSubsystem that merges SDL3Window + WindowSubsystem into a single class (per D-01, D-12).

    **Header (SDLWindowSubsystem.h):**
    - `class SDLWindowSubsystem final : public AppSubsystem` in `Wayfinder` namespace
    - Public methods directly exposing window operations (no virtual dispatch through abstract Window):
      - `Initialise(EngineContext&) -> Result<void>` override - calls SDL_Init(SDL_INIT_VIDEO | SDL_INIT_EVENTS), creates SDL_Window with config from ConfigService
      - `Shutdown()` override - destroys SDL_Window, calls SDL_QuitSubSystem
      - `PollEvents()` - SDL_PollEvent loop, translates SDL events to Wayfinder events, fires callback
      - `SetEventCallback(std::function<void(Event&)>)` - store event callback
      - `GetWidth() const -> uint32_t`, `GetHeight() const -> uint32_t`
      - `ShouldClose() const -> bool`
      - `GetNativeWindow() const -> SDL_Window*` - needed by SDLRenderDeviceSubsystem
      - `IsVSync() const -> bool`, `SetVSync(bool)`
      - `SetTitle(std::string_view)`, `SetSize(uint32_t, uint32_t)`
    - Private members: `SDL_Window* m_window`, dimensions, title, vsync, shouldClose flag, callback

    **Implementation (SDLWindowSubsystem.cpp):**
    - Merge SDL3Window::Initialise() logic + WindowSubsystem::Initialise() config reading
    - Read config via `context.GetAppSubsystem<ConfigService>().Get<EngineConfig>()` (same pattern as existing WindowSubsystem.cpp)
    - SDL3Window's event translation logic (SDL_EVENT_* -> Wayfinder Event types) moves directly into PollEvents()
    - Copy SDL3Window's destructor cleanup into Shutdown() + destructor safety check
    - Important: SDL_Init flags must include SDL_INIT_VIDEO | SDL_INIT_EVENTS (SDL3Input and SDL3Time depend on SDL being init'd, DependsOn ordering ensures Window inits first)

    **Do NOT:**
    - Include or reference Window.h (abstract base) - that's the v1 type being removed
    - Use PlatformBackend enum or BackendConfig.h
    - Create any factory method - this IS the implementation
  </action>

    - SDLWindowSubsystem.cpp contains `SDL_PollEvent` call
