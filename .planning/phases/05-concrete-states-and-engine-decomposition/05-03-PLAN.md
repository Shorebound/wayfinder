**`engine/wayfinder/src/gameplay/SceneRenderExtractor.h`**:
- Namespace: `Wayfinder`
- Class `SceneRenderExtractor` (NOT a flecs system per D-15)
- Constructor takes `flecs::world&` reference to cache queries
- Main method: `void Extract(SceneCanvas& canvas) const` - reads cached ECS queries, writes to SceneCanvas
- Optional: `void Extract(SceneCanvas& canvas, const BlendableEffectRegistry* registry) const` overload for post-process effect resolution
- No dependency on Renderer, RenderDevice, or any rendering pipeline types. Only depends on rendering vocabulary types (from RenderFrame.h) and Canvas.h.

**`engine/wayfinder/src/gameplay/SceneRenderExtractor.cpp`**:
- Adapt the extraction logic from `rendering/pipeline/SceneRenderExtractor.cpp`:
  - `ExtractViews` function: reads `ActiveCameraStateComponent` singleton, creates `RenderView`, adds via `canvas.AddView()`
  - `ExtractMeshSubmissions`: iterates entities with `TransformComponent + MeshComponent + RenderableComponent`, creates `RenderMeshSubmission`, calls `canvas.SubmitMesh()`
  - `ExtractLights`: iterates entities with `LightComponent`, creates `RenderLightSubmission`, calls `canvas.SubmitLight()`
  - `ExtractDebugDraw`: reads debug draw state, writes to canvas DebugDraw
  - Blendable effect resolution: if registry provided, blend active effects into canvas PostProcess
- Key difference from old extractor: writes to SceneCanvas methods (SubmitMesh, SubmitLight, AddView) instead of building a RenderFrame with layers
- The old `rendering/pipeline/SceneRenderExtractor.h/.cpp` is NOT removed (Phase 6/7 cleanup). Both coexist.

Add new files to `engine/wayfinder/CMakeLists.txt` in the Gameplay section.

**CRITICAL**: The old `rendering/pipeline/SceneRenderExtractor` is NOT modified or removed. The new gameplay-domain version coexists. Phase 6 switches over; Phase 7 removes the old one.
