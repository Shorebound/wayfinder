    **Create `engine/wayfinder/src/app/ApplicationStateMachine.h`** in namespace `Wayfinder`:

    Include: `OrchestrationTypes.h`, `core/Result.h`, `gameplay/Capability.h`, `<concepts>`, `<memory>`, `<span>`, `<typeindex>`, `<unordered_map>`, `<unordered_set>`, `<vector>`.

    Class `ApplicationStateMachine` (NOT derived from any base, NOT a template):

    **Public API:**
    - `template<std::derived_from<IApplicationState> T> void AddState(CapabilitySet capabilities = {})` - Register a state type with factory. Asserts not finalised, no duplicates. Creates entry with `std::make_unique<T>` factory. Per D-01, D-02, D-03.
    - `void SetInitialState(std::type_index stateType)` - Mark which state starts first.
    - `template<std::derived_from<IApplicationState> TFrom, std::derived_from<IApplicationState> TTo> void AddTransition()` - Declare a valid flat transition. Asserts not finalised.
    - `template<std::derived_from<IApplicationState> T> void AllowPush()` - Declare a state as pushable (any state can push to it). Asserts not finalised.
    - `[[nodiscard]] auto Finalise() -> Result<void>` - Validate graph and create state instances from factories. Checks: initial state registered, all transition targets registered, BFS reachability from initial state (considering both flat transitions and push targets as reachable). Returns error on failure. Sets `m_finalised = true`.
    - `void Start(EngineContext& context)` - Enter the initial state. Asserts finalised and not running. Pushes initial state onto stack, calls OnEnter, attaches IStateUI if registered. Sets `m_running = true`.
    - `template<std::derived_from<IApplicationState> T> void RequestTransition()` - Queue flat transition. Asserts finalised and running. If pending exists, log warning and overwrite (last-write-wins per Phase 1 decision).
    - `template<std::derived_from<IApplicationState> T> void RequestPush()` - Queue push transition. Same assertions.
    - `void RequestPop()` - Queue pop transition. Asserts stack size > 1 (cannot pop root state).
    - `void ProcessPending(EngineContext& context)` - Execute one pending operation at frame boundary. Uses `std::exchange(m_pending, std::monostate{})` then `std::visit` with the variant. Dispatches to private `ExecuteFlatTransition`, `ExecutePush`, or `ExecutePop`. If monostate, returns immediately.
    - `[[nodiscard]] auto GetActiveState() -> IApplicationState*` - Return top of stack state (or nullptr if not running).
    - `[[nodiscard]] auto GetActiveStateType() const -> std::type_index` - Return type_index of top of stack.
    - `[[nodiscard]] auto GetModalStack() const -> std::span<const std::type_index>` - Read-only view of the stack.
    - `[[nodiscard]] auto IsRunning() const -> bool`
    - `[[nodiscard]] auto GetBackgroundPolicy(std::type_index stateType) const -> const EffectiveBackgroundPolicy*` - Return cached policy for a suspended state (nullptr if not suspended or not found).
    - `void SetLifecycleHooks(const LifecycleHookManifest* hooks)` - Set pointer to lifecycle hook manifest (non-owning).
    - `void RegisterStateUI(std::type_index stateType, std::function<std::unique_ptr<IStateUI>()> factory)` - Register IStateUI factory for a state type. Per D-13, D-14.
    - `void Shutdown(EngineContext& context)` - Exit all states in stack (top to bottom), detach IStateUIs, reset.

    **Private members:**
    - `std::unordered_map<std::type_index, std::unique_ptr<IApplicationState>> m_states;`
    - `std::unordered_map<std::type_index, CapabilitySet> m_stateCapabilities;`
    - `std::unordered_map<std::type_index, std::unordered_set<std::type_index>> m_flatTransitions;` (from -> set of to)
    - `std::unordered_set<std::type_index> m_pushableStates;` (states that can be pushed to)
    - `std::vector<std::type_index> m_stack;` (active state at back)
    - `PendingOperation m_pending;`
    - `std::type_index m_initialState{typeid(void)};`
    - `std::unordered_map<std::type_index, EffectiveBackgroundPolicy> m_backgroundPolicies;` (cached for suspended states)
    - `std::unordered_map<std::type_index, std::unique_ptr<IStateUI>> m_stateUIs;` (instantiated UIs)
    - `std::unordered_map<std::type_index, std::function<std::unique_ptr<IStateUI>()>> m_stateUIFactories;`
    - `const LifecycleHookManifest* m_lifecycleHooks = nullptr;`
    - `bool m_finalised = false;`
    - `bool m_running = false;`

    **Private helpers (extract common lifecycle bundles per research Pitfall 5):**
    - `void EnterState(EngineContext& context, std::type_index stateType)` - Call state->OnEnter. Create and attach IStateUI if factory exists. Fire lifecycle hooks FireStateEnter (use type_index overload).
    - `void ExitState(EngineContext& context, std::type_index stateType)` - Detach IStateUI if exists (OnDetach). Call state->OnExit. Fire lifecycle hooks FireStateExit.
    - `void SuspendState(EngineContext& context, std::type_index stateType)` - Call state->OnSuspend. Call IStateUI->OnSuspend if exists.
    - `void ResumeState(EngineContext& context, std::type_index stateType)` - Call state->OnResume. Call IStateUI->OnResume if exists.
    - `void ExecuteFlatTransition(EngineContext& context, std::type_index target)` - ExitState(current) -> replace top of stack -> EnterState(target).
    - `void ExecutePush(EngineContext& context, std::type_index target)` - Compute negotiated BackgroundPolicy from current's GetBackgroundPreferences and target's GetSuspensionPolicy, cache it. SuspendState(current) -> push target onto stack -> EnterState(target).
    - `void ExecutePop(EngineContext& context)` - ExitState(top) -> pop from stack -> erase cached policy for popped state -> ResumeState(new top).

    **ProcessPending cascade order (CRITICAL - must be this exact sequence):**
    1. `auto operation = std::exchange(m_pending, std::monostate{});`
    2. `std::visit` dispatching to Execute* helpers
    3. Inside each Execute* helper, the order is: exit/suspend old state, stack manipulation, enter/resume new state (this already fires hooks and IStateUI lifecycle in the Enter/Exit/Suspend/Resume helpers)

    **Note on LifecycleHookManifest:** The existing manifest has template `FireStateEnter<TState>` which uses `typeid(TState)`. ASM works with `std::type_index` at runtime. Add a non-template overload to LifecycleHookManifest that takes `std::type_index` directly:
    ```cpp
    void FireStateEnter(EngineContext& context, std::type_index stateType) const
    {
        auto it = OnStateEnter.find(stateType);
        if (it != OnStateEnter.end())
            for (const auto& hook : it->second)
                hook(context);
    }
    ```
    Add the same for `FireStateExit`. Add these to `engine/wayfinder/src/plugins/LifecycleHooks.h`.
