    **OverlayEntry struct** (public, used by OverlayStack and tests):
    ```cpp
    struct OverlayEntry
    {
        IOverlay* Overlay = nullptr;       // non-owning
        std::type_index Type;
        CapabilitySet RequiredCapabilities;
        int32_t Priority = 0;
        bool CapabilitySatisfied = false;  // computed on UpdateCapabilities
        bool ManuallyActive = true;        // toggled via Activate/Deactivate

        [[nodiscard]] auto IsActive() const -> bool
        {
            return CapabilitySatisfied and ManuallyActive;
        }
    };
    ```

    **Class OverlayStack:**

    **Public API:**
    - `void AddOverlay(IOverlay* overlay, std::type_index type, OverlayDescriptor descriptor, int32_t registrationIndex)` - Add a non-owning entry. Compute effective priority: if `descriptor.Priority == -1`, use `registrationIndex`; else use `descriptor.Priority`. Set `ManuallyActive = descriptor.DefaultActive`. Sort `m_entries` by Priority (stable sort to preserve registration order for equal priorities).
    - `void UpdateCapabilities(const CapabilitySet& effectiveCaps, EngineContext& context)` - For each entry: recompute `CapabilitySatisfied` (true if RequiredCapabilities.IsEmpty() or effectiveCaps.HasAll(RequiredCapabilities)). If transitioning from inactive to active: call `Overlay->OnAttach(context)` (log warning on failure, set ManuallyActive = false). If transitioning from active to inactive: call `Overlay->OnDetach(context)` (log warning on failure). Per research Pattern 3.
    - `auto ProcessEvents(EngineContext& context, EventQueue& events) -> bool` - Iterate entries in REVERSE priority order (highest priority = top overlay first). For each active overlay: call `Overlay->OnEvent(context, events)`. If returns true, stop propagation and return true. If no overlay consumed events, return false.
    - `void Update(EngineContext& context, float deltaTime)` - Iterate entries in priority order. For each active overlay: call `Overlay->OnUpdate(context, deltaTime)`.
    - `void Render(EngineContext& context)` - Iterate entries in priority order (bottom-up, lowest priority first). For each active overlay: call `Overlay->OnRender(context)`.
    - `void Activate(std::type_index overlayType)` - Find entry by type, set ManuallyActive = true. Does NOT call OnAttach (that happens when capabilities are next satisfied, or immediately if already satisfied -- caller should call UpdateCapabilities or handle attach inline). Actually, per D-04 (runtime toggle), if capabilities are already satisfied, call OnAttach immediately.
    - `void Deactivate(std::type_index overlayType)` - Find entry by type, set ManuallyActive = false. If was active, call OnDetach.
    - `void DetachAll(EngineContext& context)` - Detach all active overlays (for shutdown). Iterate in reverse priority order.
    - `[[nodiscard]] auto GetEntries() const -> std::span<const OverlayEntry>` - Read-only view for debugging/inspection.
    - `[[nodiscard]] auto GetEntryCount() const -> size_t`

    **Private members:**
    - `std::vector<OverlayEntry> m_entries;` (sorted by Priority after AddOverlay)

    **Create `engine/wayfinder/src/app/OverlayStack.cpp`:**
    Implement all non-trivial methods (UpdateCapabilities, ProcessEvents, Activate, Deactivate, DetachAll). AddOverlay, Update, Render can be header-inline or in cpp (prefer cpp for methods that iterate).

    Use `Wayfinder::Log::Warn(LogEngine, ...)` for OnAttach/OnDetach failures - log and continue, don't propagate (overlays are non-critical).

    **TEST_SUITE("OverlayStack"):**

    1. **"Overlays execute in registration order for update"** - Add 3 overlays (A, B, C) with default priority. Update. Check UpdateOrder: A < B < C (registration order).
    2. **"Overlays render in registration order (bottom-up)"** - Add 3 overlays. Render. Check RenderOrder: A < B < C.
    3. **"Event processing is top-down (reverse order)"** - Add A, B, C. None consume. ProcessEvents. Check EventCalled order: C first, B second, A last (reverse registration). Use an order counter in each mock to verify.
    4. **"Event consumption stops propagation"** - Add A, B, C. B.ShouldConsumeEvents = true. ProcessEvents. C.EventCalled == true (checked first), B.EventCalled == true (consumed), A.EventCalled == false (propagation stopped). ProcessEvents returns true.
